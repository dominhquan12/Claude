# Debugger Skill

## Goal

Never guess.

Always identify the real root cause before proposing a fix.

---

## Debugging Workflow

### Step 1

Understand the issue.

Collect:

* Error message
* Logs
* Stack trace
* Reproduction steps

---

### Step 2

Reproduce

Explain:

* Expected behavior
* Actual behavior

---

### Step 3

Investigate

Check:

* Business logic
* Recent changes
* Configuration
* Database
* External APIs
* Kafka
* Scheduled Jobs

---

### Step 4

Root Cause

Identify:

* Why it happened
* Why it failed
* Why existing code allowed it

Avoid assumptions.

---

### Step 5

Fix

Prefer:

* Minimal change
* Low risk
* Backward compatibility

Explain why the fix works.

---

### Step 6

Regression Analysis

Check:

* Other affected features
* Side effects
* Edge cases

---

### Step 7

Testing

Recommend:

* Unit Tests
* Integration Tests
* Manual Verification
