# Planner Skill

## Goal

Never implement immediately.

Understand the problem first.

Produce a complete implementation plan before writing code.

---

## Planning Steps

### 1. Understand the Requirement

Summarize:

* Business requirement
* Technical requirement
* Expected behavior

Ask clarifying questions if necessary.

---

### 2. Impact Analysis

Identify:

* Controllers
* Services
* Repositories
* Entities
* DTOs
* Database
* Events
* External APIs
* Scheduled Jobs
* Kafka Consumers
* Kafka Producers

---

### 3. Risks

Identify:

* Breaking changes
* Performance impact
* Data migration
* Backward compatibility
* Security concerns

---

### 4. Database Changes

Review:

* Schema
* Migration
* Indexes
* Constraints

---

### 5. API Changes

Review:

* Request
* Response
* Validation
* Versioning

---

### 6. Testing Strategy

List:

* Unit Tests
* Integration Tests
* Regression Tests

---

### 7. Implementation Plan

Break work into phases.

Example:

Phase 1

* Database

Phase 2

* Backend Logic

Phase 3

* API

Phase 4

* Tests

---

### 8. Final Review

Before coding verify:

* Requirement understood
* Risks identified
* Files identified
* Tests planned

Only begin implementation after the plan is complete.
